Tus reuniones de Granola, ordenadas en Obsidian.

Panorama

Qué hace, en una frase

Cada vez que la ejecutas, granola-sync lee tus reuniones desde la app de Granola, las convierte a Markdown y las guarda como notas en tu vault de Obsidian — una nota por reunión, con su resumen, participantes y transcripción. Detecta lo que ya sincronizaste para no duplicar, y si editaste una reunión en Granola, actualiza la nota existente.

Hay dos formas de usarla, según quién seas:

  • Línea de comandos (granola-sync) — el flujo completo hacia Obsidian, ideal para dejar corriendo automático.
  • App con ventana (granola-notes-gui) — para personas no técnicas: exporta un archivo por reunión (.txt, o .md para Obsidian), sin tocar la terminal.

Novedades

Qué cambió en la versión 1.0

Esta versión endurece la herramienta para que corra sola sin fallar en silencio, y agrega dos cosas que faltaban: que las notas modificadas se re-sincronicen, y que el dry-run no cueste dinero.

Paso 1

Instalar

Necesitas Python 3.11 o superior, la app de escritorio de Granola instalada y con sesión iniciada al menos una vez, y un vault de Obsidian.

terminal
$ git clone https://github.com/naxbara/Granolaupdater.git
$ cd Granolaupdater
$ pip install -e ".[dev]"

Eso deja disponibles dos comandos: granola-sync (la sincronización a Obsidian) y granola-notes-gui (la app con ventana).

Paso 2

Configurar

Copia el archivo de ejemplo a config.yaml y edítalo con la ruta de tu vault. Las credenciales de Granola se detectan solas desde la app; no las pones aquí.

terminal
$ cp config.example.yaml config.yaml
config.yaml
vault_path: "C:\\Users\\TuNombre\\...\\ObsidianVault"

sync:
  include_transcripts: true
  fuzzy_threshold: 85        # 0-100, similitud de títulos
  notes_folder: "Reuniones"    # subcarpeta destino

enrichment:
  enabled: false            # ver sección 7
  api_key: "sk-ant-..."
  model: "claude-opus-4-8"

logging:
  dir: "logs"
  verbose: false

Todas las claves con su significado:

ClaveQué controla
vault_pathRuta a la raíz de tu vault de Obsidian. La única obligatoria.
sync.include_transcriptsSi incrusta la transcripción completa dentro de cada nota.
sync.fuzzy_thresholdQué tan parecido debe ser un título (misma fecha) para considerarse duplicado. 85 = 85%.
sync.notes_folderSubcarpeta del vault donde se guardan las notas. Por defecto Reuniones.
enrichment.enabledActiva la clasificación con IA vía API paga de Anthropic (ver sección 7).
enrichment.api_keyTu API key de Anthropic (solo si activas enrichment por API).
enrichment.modelModelo a usar. claude-opus-4-8, o claude-haiku-4-5 para menor costo.
logging.dirCarpeta de logs. Si es relativa, cae junto al config.yaml.
logging.verboseRegistro detallado (nivel DEBUG).
Credenciales

No pones usuario ni contraseña de Granola. La herramienta las lee automáticamente desde la app: en Windows de %APPDATA%\Granola, en macOS del Llavero. Solo necesitas haber iniciado sesión en Granola alguna vez.

Paso 3

Los cuatro modos

La sincronización tiene cuatro modos. El de por defecto es daily, pensado para correr todos los días.

daily

Sincroniza las reuniones de las últimas 24 horas. Es el modo por defecto.

historical

Importa todo desde una fecha. Úsalo la primera vez o para rellenar hacia atrás.

verify

Revisa la integridad de las notas ya sincronizadas: archivos faltantes o cuerpo vacío.

dry-run

Muestra qué crearía o actualizaría, sin escribir nada y sin gastar en API.

terminal
# Sync de las últimas 24h (modo por defecto)
$ granola-sync

# Primera importación: todo desde una fecha
$ granola-sync --mode=historical --from=2026-01-01

# Ver qué haría, sin escribir ni gastar
$ granola-sync --mode=dry-run

# Revisar integridad de las notas existentes
$ granola-sync --mode=verify
Recomendación

La primera vez corre --mode=dry-run para ver cuántas notas detecta sin tocar el vault. Luego --mode=historical --from=<fecha> para la carga inicial, y de ahí en adelante daily (idealmente programado, sección 5).

Cómo decide crear, actualizar o saltar

  • Salta si la reunión ya tiene nota (por su granola_id, o por título muy parecido en la misma fecha).
  • Actualiza la nota existente si editaste esa reunión en Granola después de la última sincronización.
  • Crea una nota nueva si la reunión no existe todavía en el vault.

Paso 4

Opciones de línea de comandos

OpciónEfecto
--mode=<m>daily · historical · verify · dry-run. Por defecto daily.
--from=<fecha>Fecha de inicio para el modo historical (formato YYYY-MM-DD).
--to=<fecha>Fecha de fin inclusive para historical. Con --from define un rango cerrado.
--config=<ruta>Usar un archivo de configuración distinto a config.yaml.
--vault=<ruta>Sobrescribir la ruta del vault sin editar el config.
--no-enrichSaltar el enrichment con IA en esta corrida.
--verbose · -vRegistro detallado en consola y en el archivo de log.

Paso 5

Programar la sincronización automática

Para que corra sola cada día sin que la ejecutes a mano.

Windows — Programador de tareas

Un instalador registra una tarea diaria no interactiva (sin ventana ni pausa, con registro a archivo). Usa una ventana móvil (3 días por defecto) para que un día saltado se recupere en la siguiente corrida — gratis gracias al dedup y la actualización de notas.

PowerShell
# Registrar la tarea diaria a las 9:00 (ventana móvil, 3 días por defecto)
> powershell -ExecutionPolicy Bypass -File scripts\install_scheduled_task.ps1 -Time 09:00

# Verificar / ejecutar ahora / quitar
> schtasks /query /tn GranolaSyncDaily
> schtasks /run   /tn GranolaSyncDaily
> powershell -ExecutionPolicy Bypass -File scripts\install_scheduled_task.ps1 -Uninstall

macOS / Linux — cron

Una entrada de cron que corre el sync diario a las 9:00:

crontab
0 9 * * *  cd /ruta/a/Granolaupdater && /usr/bin/python3 -m granola_sync --mode=daily --config config.yaml

Paso 6

Exportar a .txt o Markdown (app con ventana)

Para quien no usa la terminal. Es una aplicación con ventana que genera un archivo por reunión — .txt para leer en cualquier editor, o .md (Markdown con frontmatter, idéntico al del CLI) para Obsidian — en la carpeta que elijas. Corre en Windows y macOS.

terminal
$ granola-notes-gui
  1. Elige la carpeta de salida.
  2. Elige el rango: últimas 24 h, 7 días, un mes, todo el historial, o un rango de fechas exacto (desde–hasta; misma fecha en ambos = un solo día).
  3. Marca si quieres incluir la transcripción completa.
  4. Elige el formato: .txt (texto) o .md (Markdown para Obsidian).
  5. Presiona Exportar y al final abre la carpeta con un clic.

Los .txt usan UTF-8 con BOM en Windows para que el Bloc de notas y Word muestren bien los acentos. Los .md son UTF-8 plano, con frontmatter YAML, listos para tu vault.

GUI .md vs. CLI

La opción .md de la GUI es una exportación puntual (escribe los archivos en la carpeta que elijas). Para el flujo completo a Obsidian — duplicados, actualización de notas modificadas y enrichment — usa el CLI (granola-sync), que sincroniza a tu vault.

Paso 7

Enrichment: clasificar reuniones con IA

El enrichment usa IA para leer cada nota y agregar metadata al frontmatter — proyectos, tags y tipo de reunión — para que después puedas filtrar y navegar tus reuniones por tema en Obsidian sin leerlas. Hay dos caminos:

Opción A — Cowork Recomendado

En vez de que el script llame a la API paga, tu asistente de Cowork enriquece las notas directamente, aprovechando la suscripción que ya pagas. Sin costo por nota. Se hace con la skill enrich-vault: le pides — en lenguaje natural — que procese las notas del vault, y clasifica, agrega tags y, cuando aportan, secciones de Tareas y Seguimientos.

  • On-demand: «enriquece las reuniones de esta semana».
  • Automático: un agente programado que corre, por ejemplo, cada domingo sobre las notas nuevas.
Costo

La API de Anthropic cobra por token: cada nota enriquecida por el script son centavos de dólar, que se suman si procesas cientos. La vía Cowork usa tu suscripción de tarifa plana — sin costo marginal por nota. Todo lo demás de la herramienta (sync, exportar, dry-run) es gratis: la API de Granola no cobra.

Opción B — API integrada

Si prefieres que el propio sync lo haga, actívalo en el config con tu API key. Corre una llamada de IA por cada nota nueva o actualizada.

config.yaml
enrichment:
  enabled: true
  api_key: "sk-ant-tu-api-key"
  model: "claude-opus-4-8"   # o "claude-haiku-4-5" (más barato)

Referencia rápida

Campos del frontmatter de una nota

Cada nota sincronizada lleva un bloque YAML al inicio. Estos son sus campos:

CampoSignificado
granola_idIdentificador único de la reunión en Granola. Base de la detección de duplicados.
granola_updatedMarca de la última edición en Granola. Permite detectar cambios y re-sincronizar.
date · timeFecha y hora de la reunión.
participantsCorreos de los asistentes (si están disponibles).
durationDuración en minutos, si el evento de calendario la tiene.
projects · tags · meeting_typeAgregados por el enrichment (sección 7): proyectos, etiquetas y tipo.
enrichedMarca que la nota ya pasó por enrichment vía Cowork.

Si algo falla

  • «No pude conectarme a Granola» — abre la app de Granola e inicia sesión al menos una vez; la herramienta lee las credenciales de ahí.
  • Error 401 repetido — el token caducó; vuelve a autenticarte en la app de Granola y reintenta.
  • Se descartan notas al sincronizar — corre con --verbose y revisa el archivo de log; ahí se registra el detalle.
  • Notas viejas sin tags — las creadas antes de la 1.0 no se auto-actualizan; enriquécelas con Cowork (sección 7).
